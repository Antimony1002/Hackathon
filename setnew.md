# 新增功能
## 1.新增页面
- 新增一个页面，用于展示新功能的实现。
- 用户点击“模拟测试”时，跳转到新增的页面。

## 2.新增功能实现
- 在新增的页面中，实现新功能的逻辑。
- 新增的页面可以展示新功能的实现效果。

## 3.新增功能
- 用户可上传文件，新增的页面可以展示上传的文件。
- 接入文件上传接口，实现文件上传功能。
- 页面功能接入DeepSeek API，实现文件内容的分析和处理。
    - 调用DeepSeek API，将用户上传的文件内容发送给API。
    - API根据文件内容进行分析，提取出相关信息。
    - API根据提取出的信息，生成一套模拟卷。
    - API同时应分析不同类别题目出现的频率，根据频率调整模拟卷的难度。
- **输出**：API根据文件内容返回处理结果，展示在新增的页面中。

### 输入要求
- 用户需上传一个或多个文件，文件格式为PDF、DOCX等。
- 文件内容需包含待处理的文本信息。
- 文件为用户需要复习科目的历年真题。
- 用户需填写自己的预期分数，模拟卷的难度将根据预期分数调整。
- 用户需填写模拟卷的题目数量，模拟卷将包含用户指定数量的题目。
- 用户需填写模拟卷的题目类型，模拟卷将包含用户指定类型的题目。

### 输出要求
- API经过分析，出一套模拟卷，包含模拟卷的题目和答案。
- 模拟卷的题目和答案，根据用户上传的文件内容生成。
- 新生成的模拟卷内容应与用户上传内容相关，符合用户复习的需求。
- 模拟卷应被整理成一个文件，用户可直接下载。

### 视觉效果
- 仿照之前生成的网页的视觉效果，新增的页面保持一致的外观。
- 新增的页面包含上传文件的按钮、文件上传进度条、文件上传成功提示等元素。
- 新增的页面展示模拟卷的题目和答案，用户可直接查看和对比。

## 4.技术栈要求
- 新增的页面需使用HTML、CSS、JavaScript等技术实现。
- 新增的页面需调用DeepSeek API，实现文件内容的分析和处理。
- 新增的页面需使用Python Flask框架实现后端逻辑。
- 新增的页面需使用Tailwind CSS框架实现前端界面。
- 前端与原有网页和功能保持一致的外观和交互效果。

### 构建工具
- 使用 Next.js 作为现代化构建工具
- 支持 React、TypeScript 和服务端渲染
- 热重载开发服务器和自动优化

### 样式框架
- 使用 Tailwind CSS 作为原子化 CSS 框架
- 集成 DaisyUI 组件库
- 支持响应式设计和暗黑模式
- 新增的页面需使用DaisyUI组件库实现前端界面。

### 开发规范
- 使用语义化 HTML5 标签
- 采用 Mobile-First 设计理念
- 保证可访问性和 SEO 友好

## API调用
- 使用API sk-97e3c59e8e6646edaff3e87fd27265eb
- API用于读取文件内容，生成模拟卷。
- API调用示例：
    - 请求URL：POST https://api.deepseek.cn/v1/chat/completions
    - 请求头：
        - Content-Type: application/json
        - Authorization: Bearer sk-97e3c59e8e6646edaff3e87fd27265eb
    - 请求体：
        - model: gpt-3.5-turbo
        - messages:
            - role: user
            - content: 请根据以下文件内容生成一套模拟卷，文件内容为：...
