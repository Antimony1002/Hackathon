# 文件读取
- **内容**: 只读取文件中的文字部分
- **模型**：调用通义千问大模型读取文件内容
- **读取要求**
    - 对文档内容进行读取
    - 按照章节内容对文档进行总结
    - 生成的复习计划应与文档内容相关联
    - 不要只读标题

## 可接受文件类型
- 可接受pdf、ppt、pptx、doc、docs
- 遇见其他格式弹出“不支持该文件类型T^T”

# 文件大小
- 单个文件不超过200MB
- 文件数量不超过10个

# 文件显示情况
- **视觉要求**：参考deepseek网站中文件的显示

# 生成结果
- “一键生成复习方案”中，调用API
- **生成结果要求**：
    - 根据用户选择的天数以及上传的复习资料生成具体每日复习的内容
    - 根据不同预期分数（由用户输入）生成不同的复习计划
    - 不同的预期分数对应不同的复习计划
    - 计划包括每日复习的内容，根据用户上传的文件给出精确到章节的复习计划
    - 生成的结果整理成一个pdf文件
- **视觉效果**：
    - 在侧边栏提供一个复习pdf的大致预览，要求输出每天复习的目标
    - 在侧边栏显示生成的文件，并提供下载按钮
    - 按钮使用 px-6 py-3 rounded-lg font-medium，主按钮为蓝色实心，次要按钮为白色描边

# 文件后续处理
- **预期功能**：已经上传的文件可以删除
    - 用户点击文件末尾的叉号后可以将文件删除
    - 文件要实现可以替换
    - 已有的文件可以被删除
    - 用户可以更换文件重新生成复习方案
- **视觉效果**：
    - 点按叉号时叉号有略微浮起，实现交互效果
    - 点击叉号后对应文件栏消失

## 技术栈要求

### 构建工具
- 使用 Next.js 作为现代化构建工具

- 支持 React、TypeScript 和服务端渲染

- 热重载开发服务器和自动优化

### 样式框架

- 使用 Tailwind CSS 作为原子化 CSS 框架

- 集成 DaisyUI 组件库

- 支持响应式设计和暗黑模式

### 开发规范

- 使用语义化 HTML5 标签

- 采用 Mobile-First 设计理念

- 保证可访问性和 SEO 友好

- 使用API deepseekAPI
